# Contributing to PipeProxy 🚀

First off, thank you for considering contributing to PipeProxy! It's people like you that make PipeProxy such a great tool for the community.

## 💡 How Can I Contribute?

### Reporting Bugs
- Check the [Issues](https://github.com/TommasoRonchin/PipeProxy/issues) to see if it's already recorded.
- If not, open a new issue. Include a clear title, a description of when it happens, and steps to reproduce.

### Suggesting Enhancements
- Open an issue with the [Enhancement] tag.
- Describe the feature and why it would be useful.

### Pull Requests
1. **Fork** the repo and create your branch from `main`.
2. If you've added code that should be tested, **add tests**.
3. Ensure the **test suite passes** (`npm test`).
4. Make sure your code lints.
5. Issue that **Pull Request**!

## 🛠️ Local Development

1. **Install Dependencies**: `npm install`
2. **Run Tests**: `npm test`
3. **Run Locally**:
   - Start Server: `npm run server`
   - Start Client: `npm run client`

## 📜 Code of Conduct
By contributing to this project, you agree to abide by the terms of the [Apache 2.0 License](LICENSE).

## 💎 Design Goals
- **Zero-Dependency Core**: Keep things lightweight.
- **Security First**: No feature should compromise encrypted tunnel integrity.
- **Performance**: Use Node.js streams and multiplexing efficiently.

Thanks for helping out!
